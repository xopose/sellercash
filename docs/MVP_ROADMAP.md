# SellerCash Roadmap (MVP -> Neural Intelligence)

## 1. Стратегическая цель

Сформировать систему, которая предупреждает кассовый разрыв заранее и предлагает экономически обоснованные сценарии действий с учетом:

- внутренних финансовых данных селлера;
- внешних текстовых сигналов (документы, новости, Interfax-like источники);
- неопределенности прогноза и операционных ограничений бизнеса.

## 2. Текущее состояние (на конец MVP-1)

Реализовано:

- импорт и нормализация финансовой ленты;
- baseline cashflow forecast + сценарный расчет;
- Lucene/OpenSearch слой и rule-based extraction событий;
- релевантность по seller context;
- учет impacts в cashflow;
- UI, Keycloak, Prometheus/Grafana.

Ограничение MVP-1: прогнозный и event-контур в основном детерминированный/правиловый. Это хорошо для надежного демо, но ограничивает точность на сложных паттернах спроса и неявных текстовых сигналах.

## 3. Целевая эволюция: где именно нужна нейросеть

Нейросетевой контур добавляется не «вместо» текущей логики, а поверх нее, как слой повышения качества.

1. `Neural Event Extraction`
- модель NER + relation extraction по новостям/документам;
- извлечение атрибутов: тип события, величина, дата вступления, объект влияния (товар/страна/маршрут).

2. `Neural Relevance Scoring`
- семантическое сопоставление «внешний сигнал -> профиль конкретного селлера»;
- приоритет релевантных шоков и снижение ложных срабатываний.

3. `Neural Cashflow Forecast`
- многогоризонтный прогноз cashflow (7/14/30/60 дней) с квантилями;
- учет лагов выплат/возвратов и экзогенных признаков из knowledge слоя.

4. `Scenario Ranking Model`
- ранжирование сценариев по критериям: снижение риска, влияние на прибыль, устойчивость к стрессу.

Принцип внедрения: `fallback-first` (если нейроконтур недоступен, система работает на baseline-алгоритмах).

## 4. Дорожная карта по этапам

## Этап A (2 недели): Data & Evaluation Foundation

Цель: подготовить данные и строгий контур валидации.

- собрать размеченный набор текстовых событий (минимум 800 фрагментов);
- собрать бэктест-датасет cashflow (минимум 12 месяцев по 20+ селлерам);
- зафиксировать baseline-метрики и протоколы расчета;
- реализовать единый `evaluation` пакет (offline скрипты + отчеты).

Критерий выхода: baseline зафиксирован, есть воспроизводимый отчет качества.

## Этап B (3 недели): Neural Event Extraction + Relevance

Цель: улучшить качество обработки внешних сигналов.

- обучить модель извлечения событий (ru/eng news mix);
- внедрить neural relevance scorer с учетом seller context;
- добавить confidence-калибровку и пороги для production.

Критерий выхода: рост F1 по извлечению и precision релевантности при сохранении latency SLA.

## Этап C (3 недели): Neural Cashflow Forecast

Цель: повысить качество прогноза денежного остатка и раннего предупреждения.

- внедрить модель временных рядов (например TFT/N-HiTS/LSTM);
- добавить экзогенные признаки из impacts;

Критерий выхода: статистически значимое улучшение lead time и recall предупреждений.

## Этап D (2 недели): Scenario Intelligence

Цель: сделать рекомендации устойчивыми и проверяемыми.

- ранжирование сценариев по multi-objective функции;
- стресс-тест сценариев на «шоковых» окнах;
- объяснение рекомендации: какой фактор дал эффект.

Критерий выхода: рост доли кейсов, где система находит рабочий сценарий при ограничении по прибыли.

## Этап E (1 неделя): Production Hardening

Цель: эксплуатационная надежность.

- мониторинг model drift и качества в онлайне;
- версионирование моделей и rollback;
- SLO/SLA и runbook для инцидентов.

Критерий выхода: стабильная работа в пилоте + операционные регламенты.

## 5. Ключевые метрики результата и целевые значения

| Метрика | Формула | Целевое значение | Как подтверждается |
|---|---|---|---|
| `Event Extraction F1` | `2 * (precision * recall) / (precision + recall)` по размеченному корпусу событий | `>= 0.88` | offline evaluation на фиксированном test split + отчет артефактов |
| `Relevance Precision@1` | доля кейсов, где top-1 impact действительно релевантен seller profile | `>= 0.85` | экспертная разметка + blind review 2 аналитиками |
| `Signal-to-Impact Latency P95` | P95 времени от ingest сигнала до появления impact | `<= 5 сек` | нагрузочный прогон 500 сигналов, лог-метрики в Prometheus |
| `Cash Gap Recall@7d` | доля фактических уходов в минус, обнаруженных минимум за 7 дней | `>= 0.85` | rolling backtest на исторических срезах |
| `Median Early-Warning Lead Time` | медиана `(дата фактического разрыва - дата первого алерта)` | `>= 14 дней` | журнал алертов vs факт cash breach по бэктесту |
| `Scenario Feasibility Rate` | доля кейсов, где найден сценарий с `min_balance >= 0` и `profit_delta_pct >= -5%` | `>= 65%` | batch прогон сценариев по тестовым эпизодам |

## 6. Риски и меры контроля

- Риск переобучения нейромодели: `time-based split`, регуляризация, out-of-time тест.
- Риск деградации на новых новостных паттернах: active learning и регулярное дообучение на ошибках.
- Риск роста задержек: асинхронный inference + кэш + fallback на rule-based pipeline.
- Риск недоверия бизнеса: explainability-by-design (ссылка на источник + расшифровка влияния в деньгах).

## 7. Итоговый целевой статус

SellerCash:

- предупреждает разрыв заранее;
- объясняет причинно-следственную цепочку;
- учитывает внешние текстовые шоки;
- предлагает рабочие сценарии с контролем риска и прибыли.
